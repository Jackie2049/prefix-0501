01混合注意力架构

Qwen 3.5 引入了混合层架构：在全注意力层之间穿插线性注意力层（GatedDeltaNet）， 当前 prefix-sharing 代码假设所有层均使用标准注意力 + 传统 KV Cache。 

Qwen2 / Llama（当前支持）          Qwen 3.5 （混合架构）
─────────────────────────            ─────────────────────────────────
Layer 0  [Full Attention]             Layer 0  [Full Attention]    ← 可做 prefix sharing
Layer 1  [Full Attention]             Layer 1  [Linear Attn]    ✗ 无传统 KV Cache
Layer 2  [Full Attention]             Layer 2  [Linear Attn]    ✗ 无传统 KV Cache
Layer 3  [Full Attention]             Layer 3  [Linear Attn]    ✗ 无传统 KV Cache
Layer 4  [Full Attention]             Layer 4  [Full Attention]    ← 可做 prefix sharing
  ...     全部相同                       Layer 5  [Linear Attn]
Layer 31 [Full Attention]               ...     每 4 层循环
                                     Layer 31 [Linear Attn]

KV Cache: 所有层都参与共享         KV Cache: 仅 1/4 层可参与共享

当前模型（Qwen2）

属性值
层类型全部 Full Attention
注意力头数32 Q / 4 KV
head_dim128
RoPE全维度 (factor=1.0)
KV Cache[B, 4, seq, 128]
Prefix 层数占比32/32 = 100%

Qwen 3.5（Full Attn 层）

属性值
层类型混合（1 Full + 3 Linear/4层）
注意力头数16 Q / 4 KV
head_dim256
RoPE部分维度 (factor=0.25)
KV Cache[B, 4, seq, 256]
Prefix 层数占比8/32 = 25%

03标准注意力 — Qwen2

当前 prefix-sharing 所支持的典型注意力机制。

MATRIX FLOW Standard Multi-Head Attention (GQA) 
Input x: [B, S, 4096]
                │
    ┌───────────┼───────────┐
    │           │           │
 q_proj     k_proj     v_proj
 [4096×4096] [4096×512] [4096×512]
    │           │           │
    ▼           ▼           ▼
 Q:[B,S,32,128]  K:[B,S,4,128]  V:[B,S,4,128]
    │           │           │
    │     RoPE(factor=1.0)  │
    │           │           │
    │     K repeat_interleave(8)  ← GQA: 32/4=8
    │     V repeat_interleave(8)
    │           │           │
    ▼           ▼           │
 scores = Q @ KT / √128    │
    │                       │
    ▼                       │
 probs = softmax(scores)    │
    │                       │
    └───────────┬───────────┘
                │
                ▼
    output = probs @ V    → [B, S, 32, 128]
                │
                ▼
            reshape → [B, S, 4096]
                │
                ▼
             o_proj [4096×4096]
                │
                ▼
         Final Output: [B, S, 4096]

KV Cache: K_store [B, 4, S, 128], V_store [B, 4, S, 128]
Prefix Sharing: 复制 provider 的 K_store[:prefix_len], V_store[:prefix_len]

04Full Attention — Qwen 3.5

Qwen 3.5 的全注意力层，与 Qwen2 的关键差异已在右侧用 橙色 标注。

MATRIX FLOW Qwen 3.5 Full Attention 
Input x: [B, S, 4096]
                │
    ┌───────────┼───────────┬───────────┐
    │           │           │           │
 q_proj     k_proj     v_proj    gate_proj
 [4096×4096] [4096×1024] [4096×1024] [4096×4096]
  ↑ 输出包含 Q 和 gate     │
    │           │           │           │
    ▼           ▼           ▼           ▼
 Q:[B,S,16,256]  K:[B,S,4,256]  V:[B,S,4,256]  gate:[B,S,4096]
    │           │           │
    ▼           ▼           │
 RMSNorm(Q)  RMSNorm(K)   │
    │           │           │
    │    RoPE(factor=0.25)    │
    │    ↑ 仅 64/256 维度施加旋转  │
    │           │           │
    │     K repeat_interleave(4)  ← GQA: 16/4=4
    │     V repeat_interleave(4)
    │           │           │
    ▼           ▼           │
 scores = Q @ KT / √256    │
    │                       │
    ▼                       │
 probs = softmax(scores)    │
    │                       │
    └───────────┬───────────┘
                │
                ▼
    attn_out = probs @ V  → [B, S, 16, 256] → reshape [B, S, 4096]
                │
                ▼
    output = attn_out × sigmoid(gate)   ← 输出门控
                │
                ▼
             o_proj [4096×4096]
                │
                ▼
         Final Output: [B, S, 4096]

KV Cache: K_store [B, 4, S, 256], V_store [B, 4, S, 256]
注意：head_dim 从 128 增至 256，KV 存储量翻倍
ℹ
对 prefix-sharing 的影响：Full Attention 层完全兼容当前逻辑—— GQA repeat 和 einsum 操作是泛化的（通过 q_heads % kv_heads 动态计算）， head_dim=256 不需要改 backend 代码。唯一需确认的是 MRoPE 在纯文本场景下是否退化为标准 RoPE。

05Linear Attention — GatedDeltaNet

Qwen 3.5 独有的线性注意力机制，与传统注意力完全不同，无法参与 prefix sharing。

MATRIX FLOW Gated Delta Net (Linear Attention) 
Input x: [B, S, 4096]
                │
    ┌───────────┼───────────┬───────────┬───────────┐
    │           │           │           │           │
in_proj_qkv  in_proj_z   in_proj_b   in_proj_a
 [4096×8192] [4096×4096] [4096×32]   [4096×32]
    │           │           │           │
    ▼           ▼           ▼           ▼
 raw:[B,S,8192] z:[B,S,32,128] b:[B,S,32]  a:[B,S,32]
 split→Q,K,V     │           │           │
    │            │           │           │
    ▼            │           │           │
 Causal Conv1D(k=4)     │           │
 + SiLU 激活           │           │
    │            │           │           │
    ▼            │           │           │
 Q:[B,S,16,128]           │           │
 K:[B,S,16,128]           │           │
 V:[B,S,32,128]           │           │
    │            │           │           │
    │            │           ▼           ▼
    │            │    β = sigmoid(b)  G = -exp(A_log)×softplus(a+dt_bias)
    │            │           │           │
    ▼            │           │           │
 ── Gated Delta Rule（循环更新）──────────────────
    │            │           │           │
    │     ┌──────┘           │           │
    │     │   St = Gt ⊙ St-1 + βt × KtT @ Vt
    │     │                  │           │
    │     │   Ot = Qt @ St
    │     │                  │           │
    ▼     ▼                  │           │
 core_out:[B,S,4096]         │           │
    │                       │           │
    └───────┬───────────────┘           │
            │                           │
            ▼                           │
   output = core_out × silu(z)             │
            │
            ▼
        o_proj [4096×4096]
            │
            ▼
     Final Output: [B, S, 4096]

NO softmax(Q×K^T), NO 传统 KV Cache, 无法做 prefix sharing
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

状态缓存:
  conv_states:     [B, 8192, 3]            ← 固定大小，无 seq_len 维度
  recurrent_state: [B, 16, 128, 128]       ← 固定大小，无 seq_len 维度
✕
Prefix Sharing 不可行：Linear Attention 层没有传统 KV Cache（无 [batch, heads, seq_len, head_dim] 结构），其状态是固定大小的 conv_states + recurrent_states，不可按序列长度扩展。 这类层必须跳过 prefix sharing 流程。

06KV Cache 结构对比

三种注意力机制的缓存结构差异，直接决定 prefix sharing 的可行性。

Qwen2 标准注意力

key_cache: [B, 4, S, 128]
val_cache: [B, 4, S, 128]
S 维度随序列长度增长，可按 prefix_len 切片复用
✓ 完全兼容 prefix sharing

Qwen3.5 Full Attention

key_cache: [B, 4, S, 256]
val_cache: [B, 4, S, 256]
结构相同，head_dim 翻倍，可按 prefix_len 切片复用
✓ 兼容 prefix sharing（存储量 ×2）

Qwen3.5 Linear Attention

conv_states: [B, 8192, 3]
recurrent: [B, 16, 128, 128]
固定大小状态，无 S 维度，无法按序列切片
✗ 无法参与 prefix sharing

prefix_sharing/core/planner.py — 层过滤逻辑 MODIFY

当前 PrefixSharingPlan 假设所有层都参与 prefix sharing。 需要新增字段标识哪些层参与，以及全局层索引到 full_attention 层索引的映射。

planner.py — PrefixSharingPlan 新增字段 ADD 
# 在 PrefixSharingPlan dataclass 中新增：

active_layer_ids: list[int] = field(default_factory=list)
"""参与 prefix sharing 的全局层索引列表（仅 full_attention 层）。"""

num_active_layers: int = 0
"""active_layer_ids 的长度，即 full_attention 层的数量。"""

# 计算方式：
def _build_active_layers(layer_types: list[str]) -> list[int]:
    return [i for i, t in enumerate(layer_types) if t == "full_attention"]
# e.g. 32 层 → active_layer_ids = [0, 4, 8, 12, 16, 20, 24, 28]
# num_active_layers = 8
prefix_sharing/integrations/megatron_runtime.py — 层类型判断 + 跳过 MODIFY

这是最关键的改动：maybe_run_prefix_sharing_attention() 需要判断当前层是否为 full_attention， 如果是 linear_attention 则直接返回 None（走原始 Megatron 路径）。

megatron_runtime.py — maybe_run_prefix_sharing_attention() 修改 MODIFY 
def maybe_run_prefix_sharing_attention(
    attention_module, query, key, value,
    attention_mask, rotary_pos_emb, packed_seq_params, *,
    mscale=1.0,
):
    # ... 现有 ctx 检查 ...

    layer_id = int(getattr(attention_module, "layer_number", 0) or 0)

    # ── 新增：检查当前层是否参与 prefix sharing ──
    plan = ctx.prefix_sharing_plan
    if hasattr(plan, 'active_layer_ids') and plan.active_layer_ids:
        if layer_id not in plan.active_layer_ids:
            return None  # linear_attention 层，走原始路径
        # 将全局层 ID 映射到 active 层 ID（用于 KV store 索引）
        active_layer_id = plan.active_layer_ids.index(layer_id)
    else:
        active_layer_id = layer_id  # 非混合模型，保持原逻辑

    # ... 后续 RoPE + build_kv + attention 逻辑 ...
    # 将 layer_id 替换为 active_layer_id 传入 build_kv
    expanded_key, expanded_value = backend.build_kv(
        key, value, ctx.store, ctx.prefix_sharing_plan,
        layer_id=active_layer_id,  # ← 改动点
        tp_rank=tp_rank,
    )
prefix_sharing/backends/torch_ref.py — 无需改动（验证通过） NO CHANGE

当前 backend 的 GQA repeat 和 attention 计算是泛化的，不需要改动。 关键代码如下：

torch_ref.py L162-174 — GQA 逻辑已是动态的 
# 已通过 q_heads % kv_heads 动态计算 repeat 比例
# Qwen2:   32 / 4 = 8 次重复
# Qwen3.5: 16 / 4 = 4 次重复
# head_dim=128 或 256 都不影响 einsum 操作

q_heads = q_row.shape[1]   # 自动获取
kv_heads = k_row.shape[1]  # 自动获取
if q_heads != kv_heads:
    repeat = q_heads // kv_heads
    k_row = k_row.repeat_interleave(repeat, dim=1)
    v_row = v_row.repeat_interleave(repeat, dim=1)

scores = torch.einsum("qhd,khd->hqk", q_row, k_row) / scale
# ↑ scale = sqrt(head_dim)，自动适配 128 或 256
✓
无需改动：_attention_row() 中的 GQA repeat 使用 q_heads // kv_heads 动态计算，scale = sqrt(q_row.shape[-1]) 自动适配不同 head_dim。build_kv() 中的 KV cat 操作也是形状无关的。
