1. 一句话结论

PR #6 的工程脚手架（factory / mixin / tests / docs）非常优秀，但其 GPU 后端在 reuser 场景下数学上不正确。 原因：flash_attn_varlen_func(causal=True) 使用段内相对位置的 causal mask，而 prefix sharing 需要绝对位置的 block-causal mask（prefix 列全可见 + suffix causal）。 融合方向：保留 PR #6 的全部工程骨架，替换 GPU/NPU 两个 attention() 实现为 attention_bias 形态，并新增统一的 mask 构造模块。

3. PR #6 的关键正确性问题

3.1 问题描述

PR #6 的 GPU 后端核心调用：

out = flash_attn_varlen_func(
    q, k, v,
    cu_seqlens_q, cu_seqlens_kv,
    max_seqlen_q, max_seqlen_kv,
    causal=True,        # ⚠️ 关键问题
    ...
)

flash_attn_varlen_func 在 causal=True 时，对每个 segment 独立应用段内相对位置的下三角 mask：
query 在 segment 内位置 i 只能看 key 在 segment 内位置 0..i。

但 prefix sharing 的 reuser 场景需要绝对位置的 block-causal mask：

Reuser: Q 是 suffix（绝对位置 P..P+L-1），KV = prefix(P) + suffix(L) 期望的 attention mask（行=Q相对位置，列=KV相对位置）: KV: 0 1 2 ... P-1 | P P+1 ... P+L-1 Q 0 : [1 1 1 ... 1 | 1 0 0 ... 0 ] ← 看 prefix 全部 + suffix[0] Q 1 : [1 1 1 ... 1 | 1 1 0 ... 0 ] ← 看 prefix 全部 + suffix[0..1] ... Q L-1 : [1 1 1 ... 1 | 1 1 1 ... 1 ] FA causal=True 实际给出的 mask（错误）: KV: 0 1 2 ... P-1 | P P+1 ... P+L-1 Q 0 : [1 0 0 ... 0 | 0 0 0 ... 0 ] ← 只看 KV[0]，错过 prefix[1..] Q 1 : [1 1 0 ... 0 | 0 0 0 ... 0 ] ... Q L-1 : [1 1 1 ... 1 | 0 0 0 ... 0 ] ← suffix KV 全部被屏蔽！

对于 reuser，cu_seqlens_kv 包含 P+L 个 token（prefix+suffix），但 cu_seqlens_q 只有 L 个。FA 在段内对齐 Q 和 KV 的相对位置 0..L-1，对应到 KV 上是 0..L-1（即 prefix 部分），suffix 部分（KV 的 P..P+L-1）完全被屏蔽。

3.2 为什么 PR #6 的测试没发现？

test_flash_atten_gpu_vs_torch_ref 用 atol=5e-2（fp16）和 atol=2e-1（grad）。这个精度太松：
  • 测试输入用 torch.randn(...) * 0.02，本身数值很小，softmax 后概率近似均匀，attention 输出几乎是 value 的简单平均。
  • 当 mask 错误地把一些位置置零，输出差异在 1e-2 量级，仍能通过 5e-2 阈值。
  • 测试用例 batch_sizes=[8,8], prefix_lens=[0,4] 包含一个 reuser，但 prefix 占比只有 50%，差异被稀释。
结论：当前的 numerical alignment 测试不足以验证 mask 正确性，需要显式 mask 断言。

5. 融合方案（文件级）

5.1 总原则

以 PR #6 为基线合并，把多硬件方案的「正确性内核」嵌入 PR #6 的工程骨架。
  • 不另起炉灶，避免冲突 PR #6 已有的 12 个文件改动
  • 在 flash_atten_base.py 新增 build_block_causal_mask() 静态方法，作为 mixin 的一部分
  • GPU 后端：新增 TE 路径，降级 flash_attn 路径（仅 provider-only 场景可用，否则报错）
  • NPU 后端：从占位符变为真实实现

5.3 关键代码：统一 mask 构造

把 mask 构造放到 flash_atten_base.py，作为 mixin 的静态方法（PR #6 已经有 mixin 类，这是最低侵入位置）：

# flash_atten_base.py 新增

@staticmethod
def build_block_causal_mask(
    plan: PrefixSharingPlan,
    device: torch.device,
) -> torch.Tensor:
    """构造 (total_q, total_kv) 的 BoolTensor。
    True = masked (不可见), False = visible.
    统一所有 backend 的 mask 真相源。
    """
    total_q = plan.cu_seqlens_q[-1]
    total_kv = plan.cu_seqlens_kv[-1]
    mask = torch.ones(total_q, total_kv, dtype=torch.bool, device=device)

    for i in range(plan.batch_size):
        q_lo = plan.cu_seqlens_q[i]
        q_hi = plan.cu_seqlens_q[i + 1]
        kv_lo = plan.cu_seqlens_kv[i]
        kv_hi = plan.cu_seqlens_kv[i + 1]
        q_len = q_hi - q_lo
        kv_len = kv_hi - kv_lo

        if plan.is_reuser(i):
            p = plan.prefix_lens[i]
            # prefix columns: all visible
            mask[q_lo:q_hi, kv_lo:kv_lo + p] = False
            # suffix columns: causal
            for qi in range(q_len):
                mask[q_lo + qi, kv_lo + p:kv_lo + p + qi + 1] = False
        else:
            # provider: standard causal
            for qi in range(q_len):
                mask[q_lo + qi, kv_lo:kv_lo + qi + 1] = False
    return mask

@staticmethod
def mask_to_te_bias(mask: torch.Tensor, dtype: torch.dtype) -> torch.Tensor:
    """Bool mask → TE 接受的 float bias. False=0, True=-inf."""
    return torch.where(
        mask,
        torch.full_like(mask, float('-inf'), dtype=dtype),
        torch.zeros_like(mask, dtype=dtype),
    )

5.4 关键代码：GPU 后端替换

# flash_atten_gpu.py (替换 PR #6 的 attention 方法)

def attention(self, query, key, value, plan, **kwargs):
    self._validate_plan_for_flash(plan)

    # 决策：如果所有样本都是 provider，走 fast path
    has_reuser = any(plan.is_reuser(i) for i in range(plan.batch_size))

    if has_reuser:
        return self._attention_via_te(query, key, value, plan, **kwargs)
    else:
        return self._attention_via_flash_attn(query, key, value, plan, **kwargs)

def _attention_via_te(self, q, k, v, plan, **kwargs):
    import transformer_engine.pytorch as te
    from transformer_engine.pytorch.attention import DotProductAttention

    bool_mask = self.build_block_causal_mask(plan, device=q.device)
    bias = self.mask_to_te_bias(bool_mask, dtype=q.dtype)

    # 注意：TE 的 THD 路径要求 q/k/v 是 (1, total, heads, dim) + cu_seqlens
    q4 = q.unsqueeze(0)
    k4 = k.unsqueeze(0)
    v4 = v.unsqueeze(0)
    cu_q = torch.tensor(plan.cu_seqlens_q, device=q.device, dtype=torch.int32)
    cu_kv = torch.tensor(plan.cu_seqlens_kv, device=q.device, dtype=torch.int32)

    dpa = DotProductAttention(
        num_heads=..., gqa_kernel=True,
        core_attention_bias_type="post_scale_bias",
    )
    out = dpa(q4, k4, v4,
              core_attention_bias=bias,
              cu_seqlens_q=cu_q, cu_seqlens_kv=cu_kv,
              max_seqlen_q=plan.max_seqlen_q,
              max_seqlen_kv=plan.max_seqlen_kv,
              attention_mask_type="padding_causal")
    return out.squeeze(0)

def _attention_via_flash_attn(self, q, k, v, plan, **kwargs):
    # 仅 provider-only 场景，FA causal=True 与绝对 causal 等价
    q, k, v, cu_q, cu_kv, max_q, max_kv = self._prepare_flash_inputs(q, k, v, plan)
    fa_varlen = _import_flash_attn_varlen()
    return fa_varlen(q, k, v, cu_q, cu_kv, max_q, max_kv,
                     causal=True, **kwargs)

5.5 关键代码：NPU 后端从占位符到实现

# flash_atten_npu.py (替换 PR #6 的 13 行占位符)

from prefix_sharing.backends.flash_atten_base import (
    FlashAttentionMixin, FlashBackendValidationError,
)
from prefix_sharing.backends.torch_ref import TorchReferenceBackend
from prefix_sharing.backends.base import BackendCapabilities

class NpuFlashAttentionBackend(FlashAttentionMixin):
    capabilities = BackendCapabilities(
        name="flash_atten_npu",
        supports_cpu=False,
        supports_cuda=False,
        supports_cann=True,
        supports_different_q_kv_lengths=True,
        supports_prefix_last_restore=True,
        supports_flash_attention=True,
    )

    def __init__(self):
        self._torch_ref = TorchReferenceBackend()

    def validate(self, config, model_config=None):
        config.validate(model_config=model_config)
        try:
            from mindspeed.ops import npu_fusion_attention
        except ImportError as e:
            raise RuntimeError("NpuFlashAttentionBackend requires MindSpeed") from e

    def apply_rope(self, *args, **kwargs):
        return self._torch_ref.apply_rope(*args, **kwargs)

    def build_kv(self, *args, **kwargs):
        return self._torch_ref.build_kv(*args, **kwargs)

    def attention(self, query, key, value, plan, **kwargs):
        self._validate_plan_for_flash(plan)
        from mindspeed.ops import npu_fusion_attention

        q, k, v, _, _, _, _ = self._prepare_flash_inputs(
            query, key, value, plan, attention_mask=kwargs.get("attention_mask")
        )
        num_heads = q.shape[1]
        atten_mask = self.build_block_causal_mask(plan, device=q.device)

        out, _, _, _, _, _, _ = npu_fusion_attention(
            q, k, v, num_heads, input_layout="TND",
            atten_mask=atten_mask,
            actual_seq_qlen=list(plan.cu_seqlens_q[1:]),
            actual_seq_kvlen=list(plan.cu_seqlens_kv[1:]),
            scale=kwargs.get("softmax_scale"),
            keep_prob=1.0 - kwargs.get("dropout_p", 0.0),
            sparse_mode=0,
        )
        return out

5.6 关键代码：新增显式 mask 测试

# tests/unit_test/test_block_causal_mask.py

def test_block_causal_mask_reuser_sees_full_prefix():
    """关键断言：reuser Q[0] 能看到 KV 中所有 prefix token 和 suffix[0]。"""
    plan = _make_plan(batch_sizes=[4, 4], prefix_lens=[0, 2])
    # sample 0: provider, q_len=4, kv_len=4
    # sample 1: reuser,   q_len=2, kv_len=4 (prefix=2 + suffix=2)

    mask = FlashAttentionMixin.build_block_causal_mask(plan, device=torch.device("cpu"))

    # sample 1 的 reuser: q 在 mask 行 [4:6], kv 在列 [4:8]
    reuser_q0_row = mask[4, 4:8]  # q[0] 对所有 KV 的可见性
    expected = torch.tensor([False, False, False, True])  # P P S0 S1
    #                                  prefix[0,1] + suffix[0]  可见, suffix[1] 不可见
    assert torch.equal(reuser_q0_row, expected), \
        f"Expected [P P S0 !S1], got {reuser_q0_row}"

def test_block_causal_mask_provider_is_standard_causal():
    plan = _make_plan(batch_sizes=[3], prefix_lens=[0])
    mask = FlashAttentionMixin.build_block_causal_mask(plan, device=torch.device("cpu"))
    expected = torch.tril(torch.ones(3, 3, dtype=torch.bool), diagonal=0)
    expected = ~expected  # True = masked
    # 我们这里 mask=True 表示屏蔽；tril 上三角=True
    assert torch.equal(mask, expected)
这个 mask 单元测试是最关键的回归保护。它的存在让 PR #6 类型的错误（误用 FA causal=True）在 CI 阶段就能被发现，而不是依赖 numerical alignment 的 atol 容忍度。

7. 风险与回滚

风险影响缓解
TE 未安装（如 ROCm/老 CUDA）GPU 后端无法走 TE 路径检测 TE import，若不可用且存在 reuser，抛 FlashBackendValidationError 引导用户切回 torch_ref
npu_fusion_attention 接口在不同 MindSpeed 版本签名不同NPU 后端兼容性在 validate() 中尝试调用一次空 shape 探测签名
TE THD 路径在某些 Megatron-LM-core 版本要求 padding_causal，部分版本要求 causalTE 集成失败用 dependency/Megatron-LM-core_v0.12.1 当前快照锁定 padding_causal
用户已基于 PR #6 当前形态写了下游代码API 兼容性backend 名 "flash_atten_gpu" 不变；只是内部实现切换
性能：TE 路径在 provider-only 场景可能比 FA 慢性能回退provider-only 走 FA 兜底；只在 has_reuser 时走 TE
合并决议建议：接受 PR #6 的全部工程改动作为基线，在 merge 前/后立即提交一个补丁 PR 完成 §5.3–§5.6 的替换。
这样既尊重了 PR #6 作者的工程贡献（factory / mixin / 测试 / 文档），又把数学正确性问题以最小侵入方式修复。
