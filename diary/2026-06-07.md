### 全注意力重写 ✅

**发现**: 预训练模型的全注意力层与我们的实现严重不匹配:
- HF q_proj: (12288, 5120) = num_heads × head_dim × **2** (不是我们的 6144)
- HF 有 q_norm.weight 和 k_norm.weight (per-head RMSNorm, 我们的没有)
- 参考实现: q_proj → chunk → query + gate → q_norm(query) → k_norm(key) → RoPE → attention → output * sigmoid(gate) → o_proj
- 我们的旧实现: QKVParallelLinear → RoPE → attention → o_proj → sigmoid(gate_proj)

**重写要点**:
1. q_proj: ColumnParallelLinear, output = num_heads × head_dim × 2 = 12288 (fused query + gate)
2. chunk(q_proj_output, 2) → query_states + gate (替代独立 gate_proj)
3. q_norm/k_norm: RMSNormPerHead (head_dim=256, weight init zeros, matching reference)
4. RoPE AFTER q_norm/k_norm (不是之前)
5. output gate: attn_output × sigmoid(gate) BEFORE o_proj (不是之后)
6. k_proj/v_proj: ColumnParallelLinear, 无 bias
7. scaling = head_dim^-0.5 (显式指定)

**验证**: TP=4 bf16 前向传播 OK, Output (2,32,248320), GPU 14.80 GB
- verl qwen3_6 原代码: `head_dim = hidden_size // num_heads` → 5120/24 = 213.33 ❌
- Qwen3.6 配置: `head_dim = 256` (显式指定, 不等于 hidden_size/num_heads)
- **修复**: 修改 attention 和 deltanet 用 `getattr(config, "head_dim", None) or hidden_size // num_heads`
- 修复 reshape 操作用 `q_output_size_per_tp` (num_heads_per_tp * head_dim) 而非 hidden_size // tp_size
- 修复 assertion: 当 config.head_dim 显式指定时跳过 hidden_size % num_heads 检查

### DeltaNet 维度适配 ✅ → 完整重写 ✅

**发现**: 最初的 cumsum 实现与官方 Qwen3_5GatedDeltaNet 参考严重不匹配:
- ❌ 旧: q_proj → num_heads × head_dim = 24×256 = 6144, 应用 RoPE, cumsum 线性注意力, sigmoid gate_proj
- ✅ 新: in_proj_q → num_k_heads × head_k_dim = 16×128 = 2048, 无 RoPE, causal conv1d + chunk_gated_delta_rule, RMSNormGated (SiLU)

**关键架构差异** (参考 transformers/models/qwen3_5/modeling_qwen3_5.py):
1. **无 RoPE** — DeltaNet 用 causal conv1d (depthwise, kernel_size=4, SiLU) 做位置编码
2. **Fused QKV → conv1d → split** — in_proj_qkv 映射 hidden→key_dim×2+value_dim (10240), conv1d 后 split
   我们用 separate q/k/v projections (TP 分片更简单), concat 后 apply conv1d
3. **Query 用 head_k_dim (128)** — 不是 head_dim (256), 然后 GQA expand 3× (48/16) 匹配 v_heads
4. **Decay**: g = -exp(A_log) × softplus(a + dt_bias), 不是简单 sigmoid(decay_proj)
5. **Beta**: in_proj_b (per v_head), 不是 per attention_head
6. **Output**: RMSNormGated(core_attn_out, z) with SiLU gate, 不是 sigmoid(gate_proj)
7. **out_proj**: value_dim (6144) → hidden_size (5120), 不是 num_heads×head_dim → hidden_size
8. **所有 projection 无 bias** (匹配参考)
9. **torch_chunk_gated_delta_rule** 纯 PyTorch fallback (chunk_size=64, L2 norm, float32 精度)

**修复**: conv1d dtype (bf16 input vs float32 weight → 用 F.conv1d 直接 cast weight)

**验证**: 单GPU TP=1 测试通过, 输出 cos_sim=0.002 (完全不是 identity), 所有新属性存在

**权重名映射**:

### Prefix-sharing 集成 ✅ (初步)

**verl_attention.py** 重写:
- 全注意力层 patch: 支持 Qwen3.6 新架构
  - Separate q_proj/k_proj/v_proj (不再用 fused qkv_proj)
  - q_proj chunk → query + gate (fused output)
  - q_norm/k_norm (per-head RMSNorm) before RoPE
  - Partial RoPE (只对前 64/256 维应用)
  - Output gate: attn_output * sigmoid(gate) BEFORE o_proj
- DeltaNet patch: 暂时 passthrough (DeltaNet PS 待实现)
- VerlQwen3_6Integration 类: 自动 patch 安装/卸载

**model_spec.py** 路由修复:
- 旧: layer_idx % interval == 0 → layer 0 是全注意力 (错!)
- 新: (layer_idx + 1) % interval == 0 → layer 3,7,11,... 是全注意力 (对!)
- QWEN3_6_27B preset: 16 full attn + 48 linear attn

**核心发现**: DeltaNet (48层) 没有 KV cache, prefix-sharing 主要收益来自 16 层全注意力
- 全注意力层比例: 16/64 = 25%, 但这些层的 KV cache 占用最大
- GRPO n=8 模式: 每个 request 的前缀只通过全注意力层复用 KV

### PS Attention 层级精度对齐 ✅

单层全注意力 (layer 3) PS 精度测试 **PASS**:
- 总体 cos_sim: 0.999997 (阈值: 0.999)
- 总体 max_diff: 0.001953 (阈值: 0.01, bf16 容差)
- Provider (seq 0): cos_sim=0.999996
- Reuser (seq 1-3): cos_sim=0.999997-0.999998

**修复链**: prefix_sharing import path → model_type config → sin slice typo → GQA expand dims → cu_seqlens dtype (cumsum upcast int32→int64)

**关键理解**: prefix hidden_states 必须完全相同才能 PS 精度对齐。随机 hidden_states 测试需要确保所有序列共享相同的前缀 hidden_states。在真实 GRPO 场景中,同一 prompt 的 embedding 自然相同,所以自动满足。
