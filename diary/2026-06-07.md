- architecture: `Qwen3_5ForConditionalGeneration` (不是 `Qwen3ForCausalLM`)

## 模型适配进展 (选项 C 实施)

### apex fallback ✅
- verl Megatron RMSNorm 原来硬依赖 apex (`fused_rms_norm_affine`)
- 服务器没有 apex (编译需要时间)
- **解决**: 给 qwen2 和 qwen3_6 的 `parallel_rmsnorm.py` 加了 try/except fallback
- fallback 用纯 PyTorch: `x * rsqrt(mean(x^2) + eps) * weight`, 数学完全等价
- 所有 import 测试通过 ✅

### 纯文本 config 创建 ✅
- transformers 4.57.1 不认 `qwen3_5` model_type (KeyError)
- **解决**: 从多模态 config.json 的 `text_config` 提取, 创建 `Qwen3-27B-text-only` 目录
- 新 config 使用 `qwen3` model_type, transformers 可以正常加载为 `Qwen3Config`
- 保留所有 HybridAttention 参数 (layer_types, head_dim, partial_rotary_factor 等)
- 权重 shard 文件 symlink 到原始多模态目录 (节省磁盘空间)
- 权重索引只引用 language_model 权重 (866 tensors), 跳过 vision (333 tensors)

### head_dim 修复 ✅
- verl qwen3_6 原代码: `head_dim = hidden_size // num_heads` → 5120/24 = 213.33 ❌
- Qwen3.6 配置: `head_dim = 256` (显式指定, 不等于 hidden_size/num_heads)
- **修复**: 修改 attention 和 deltanet 用 `getattr(config, "head_dim", None) or hidden_size // num_heads`
- 修复 reshape 操作用 `q_output_size_per_tp` (num_heads_per_tp * head_dim) 而非 hidden_size // tp_size
- 修复 assertion: 当 config.head_dim 显式指定时跳过 hidden_size % num_heads 检查

### DeltaNet 维度适配 (进行中)
- DeltaNet 使用独立的 k/v 维度 (不同于全注意力):
  - k: linear_num_key_heads (16) × linear_key_head_dim (128) = 2048
  - v: linear_num_value_heads (48) × linear_value_head_dim (128) = 6144
  - q: num_attention_heads (24) × head_dim (256) = 6144
- k_proj/v_proj 的 output_size 已更新为线性注意力维度
- 需要进一步验证 einsum 计算与 k/v 不同 head count/dim 的兼容性

### 模型实例化测试
- TP=1, 单GPU, FP32: OOM (22.58GB/23.5GB)
- TP=1, 单GPU, bf16: params_dtype 仍是 float32 → OOM
- TP=8: `num_query_groups(4) % TP_SIZE(8) != 0` → GQA KV heads 不够分
- TP=4: 正在测试 (4 KV heads ÷ 4 GPUs = 1 head/GPU, 刚好满足)
- **关键约束**: GQA 4 KV heads → 最大 TP=4 (不是8)
- 27B bf16 ≈ 54GB, TP=4 → 每GPU ~7GB weights, 加 activations 可能需要 gradient checkpointing
